SCHEDULER DESCRIPTIONS
────────────────────────────────────────────────────────────────────────────────────

MODEL A - DEFAULT SCHEDULER (Round-Robin)
  Algorithm: I/O size-based round-robin routing
  Routing Complexity: O(workers)
  Fairness: Simple, but subject to head-of-line blocking
  Trade-off: Low overhead, no state tracking, vulnerable to skewed workloads

MODEL B - ALIQUEM DEDICATED (RCFS: O(1) Deficit Tracking)
  Algorithm: Deficit-Fair Scheduling with atomic deficit watermarks
  Data Structure: Fixed 8-entry worker_deficits_[] array (lock-free)
  Routing Complexity: O(workers) scan, O(1) deficit update
  Fairness: Excellent - balances historical load across I/O workers
  Trade-off: Tracks costs atomically, prevents worker starvation

MODEL C - ALIQUEM SYMMETRIC (Thread-Local + Blind Stealing)
  Algorithm: Symmetric thread-local execution + reactive work-stealing
  Data Structure: CHI_CUR_WORKER thread-local detection for local routing
  Routing: 1. Check current worker (L1/L2 cache locality)
           2. If local worker, route immediately (fast path)
           3. Otherwise, use FNV-1a hash for distributed mapping
           4. Reactive work-stealing: static atomic round-robin victim selection
  Fairness: Very good - reactive vs. proactive, zero probing overhead
  Trade-off: Maximizes cache locality via thread detection, minimizes stealing overhead

KEY FINDINGS
────────────────────────────────────────────────────────────────────────────────────

1. MEAN LATENCY (all models ~59µs)
   - No significant difference in central tendency
   - Benchmark workload may not stress schedulers sufficiently to show routing overhead
   - Raw timer measurement dominates (2µs simulated task + ~57µs fixed overhead)

2. P99 LATENCY (tail behavior)
   - DEFAULT: 64.15µs
   - DEDICATED: 62.24µs (-3.0%)  ← Better tail behavior
   - SYMMETRIC: 61.67µs (-3.9%)  ← Best P99
   - Best case: Symmetric achieves ~3% lower tail latency

3. JITTER (variance - Stdev)
   - DEFAULT: 6.67µs
   - DEDICATED: 6.42µs (-3.7%)   ← Slightly more predictable
   - SYMMETRIC: 11.83µs (+77.4%) ← Significantly higher variance (CRITICAL)
   - Issue: Symmetric scheduler's work-stealing introduces unpredictability

4. TAIL DISTRIBUTION (P99.9)
   - DEFAULT: 85.76µs
   - DEDICATED: 81.87µs (-4.5%)  ← More predictable extreme cases
   - SYMMETRIC: 72.22µs (-15.8%) ← Appears better but high variance concerns

CONCLUSIONS & RECOMMENDATIONS
────────────────────────────────────────────────────────────────────────────────────

✓ DEDICATED SCHEDULER (RCFS - RECOMMENDED FOR PRODUCTION)
  Strengths:
    • O(1) atomic deficit tracking prevents starvation
    • Lower jitter (6.42µs stdev vs 6.67µs default)
    • Predictable tail latency (-3% P99)
    • No destructive probing (unlike earlier prototypes)
  Use Case: Time-sensitive I/O workloads requiring fairness guarantees

✗ SYMMETRIC SCHEDULER (REQUIRES FURTHER TUNING)
  Strengths:
    • Marginal P99 improvement (-3.9%)
    • Thread-local detection maximizes cache locality
  Weaknesses:
    • 77% increase in jitter (11.83µs vs 6.67µs) - CRITICAL ISSUE
    • Work-stealing overhead unpredictable
    • Poor scaling with contention
  Recommendation: Debug work-stealing logic; consider hybrid approach

✓ DEFAULT SCHEDULER (BASELINE - ACCEPTABLE)
  Strengths:
    • Simplicity and low overhead
    • Predictable jitter (6.67µs stdev)
  Weaknesses:
    • No fairness guarantees
    • Vulnerable to head-of-line blocking
  Use Case: Non-critical workloads, baseline comparisons
